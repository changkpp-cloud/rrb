# สรุปข้อมูลโครงการสำหรับทำสไลด์: หรีดร่วมบุญ

## 1. ชื่อโครงการ

**หรีดร่วมบุญ (Hreed Ruam Bun)**  
แพลตฟอร์มหรีดดิจิทัล เปลี่ยนพวงหรีดเป็นทุน ร่วมบุญเจ้าภาพ ลดขยะต้นทาง สร้างสวัสดิการผู้วายชนม์

## 2. แนวคิดหลัก

หรีดร่วมบุญเป็นแพลตฟอร์มสำหรับงานศพไทยพุทธ ที่เปลี่ยนการมอบพวงหรีดจากสิ่งของที่ใช้ครั้งเดียวแล้วกลายเป็นขยะ ให้เป็นการร่วมทำบุญผ่านระบบดิจิทัล พร้อมแสดงรายชื่อผู้ร่วมบุญบนบอร์ดหน้างานและส่งการ์ดขอบคุณกลับให้ผู้ร่วมทำบุญ

โครงการนี้ช่วยให้เจ้าภาพจัดการรายชื่อผู้ร่วมบุญได้ง่าย ศูนย์หรือผู้ดูแลงานตรวจสอบสลิปและพิมพ์ป้ายได้เป็นระบบ และผู้ร่วมทำบุญสามารถร่วมแสดงความอาลัยได้สะดวกผ่านมือถือ

## 3. ปัญหาที่ต้องการแก้

- พวงหรีดจริงจำนวนมากกลายเป็นขยะหลังจบพิธี
- เจ้าภาพต้องจัดการรายชื่อผู้ส่งหรีดและยอดร่วมทำบุญด้วยมือ
- การตรวจสลิป การบันทึกยอด และการพิมพ์ป้ายหน้างานมีโอกาสผิดพลาด
- ผู้ร่วมงานบางคนไม่สะดวกเดินทาง แต่ยังต้องการแสดงความอาลัย
- ศูนย์หรือผู้ดูแลงานต้องการระบบที่ติดตามสถานะงานได้ชัดเจน

## 4. วิธีแก้ปัญหา

แพลตฟอร์มสร้างหน้ารำลึกเฉพาะงานศพแต่ละงาน มี QR สำหรับรับเงินร่วมทำบุญ ระบบแนบสลิป ระบบตรวจสอบโดยเจ้าหน้าที่ ระบบพิมพ์ป้ายชื่อผู้ร่วมบุญ และหน้าการ์ดขอบคุณหลังร่วมทำบุญ

การทำงานทั้งหมดออกแบบให้เป็น mobile-first สำหรับผู้ใช้งานชาวไทย โดยใช้โทนสีครีม ทอง และเขียวมรกต เพื่อให้เหมาะกับบริบทงานอาลัยและยังใช้งานง่ายสำหรับผู้สูงอายุ

## 5. กลุ่มผู้ใช้งานหลัก

**ผู้ร่วมทำบุญ**  
เปิดหน้ารำลึก เลือกร่วมทำบุญ โอนเงิน แนบสลิป กรอกชื่อสำหรับป้าย และรับ e-card หรือใบยืนยันการร่วมบุญ

**เจ้าภาพ**  
ดูสรุปงาน รายชื่อผู้ร่วมบุญ เอกสารเจ้าภาพ และข้อมูลปิดงาน

**ศูนย์หรือผู้ดูแลงาน**  
สร้างงาน เปิด/ปิดงาน ตรวจสลิป พิมพ์ป้ายชื่อ ติดตามรายการค้าง และจัดการงานประจำวันของศูนย์

**ผู้ดูแลระบบกลาง**  
ดูภาพรวมข้ามศูนย์ รายงาน ESG รายงานการเงิน สถานะระบบ ผู้ใช้ ศูนย์ และ audit log

## 6. Flow ผู้ร่วมทำบุญ

1. เข้าหน้า `/{slug}` เพื่อดูข้อมูลผู้วายชนม์และกำหนดการ
2. ไปหน้า `/{slug}/payment` เพื่อดู QR/เลขบัญชี โอนเงิน และแนบสลิป
3. ไปหน้า `/{slug}/verifying` หลังแนบสลิป เพื่อเข้าสู่สถานะตรวจสอบ
4. ไปหน้า `/{slug}/print-name` เพื่อกรอกชื่อหรือองค์กรสำหรับป้าย
5. ไปหน้า `/{slug}/ecard` เพื่อรับการ์ดขอบคุณ ใบยืนยัน และภาพจำลองการมอบหรีด

## 7. ฟีเจอร์สำคัญ

- หน้ารำลึกเฉพาะแต่ละงานศพ
- QR PromptPay และข้อมูลบัญชีสำหรับรับร่วมทำบุญ
- อัปโหลดสลิปโอนเงิน
- ระบบตรวจสอบสลิปโดยคน ไม่ยืนยันเงินอัตโนมัติจาก AI
- ระบบกรอกชื่อและพิมพ์ป้ายหน้างาน
- e-card ขอบคุณและใบยืนยันการร่วมบุญ
- AI photo flow สำหรับภาพจำลองการมอบหรีด
- Dashboard ศูนย์สำหรับงานประจำวัน
- Dashboard เจ้าภาพสำหรับดูสรุปและรายชื่อ
- Dashboard Admin สำหรับภาพรวม รายงาน ESG ผู้ใช้ ศูนย์ และ audit

## 8. จุดเด่นของโครงการ

- ลดขยะจากพวงหรีดจริงและสนับสนุนแนวคิด zero waste
- เหมาะกับบริบทงานศพไทยพุทธ ทั้งภาษา ภาพลักษณ์ และขั้นตอนใช้งาน
- แยกบทบาทชัดเจนระหว่างผู้ร่วมทำบุญ เจ้าภาพ ศูนย์ และแอดมินกลาง
- ออกแบบให้ใช้งานง่ายบนมือถือเป็นหลัก
- รองรับงานปฏิบัติจริง เช่น ตรวจสลิป พิมพ์ป้าย ปิดงาน และทำรายงาน
- ให้ความสำคัญกับความปลอดภัยของข้อมูลและการยืนยันธุรกรรมโดยมนุษย์

## 9. เทคโนโลยีที่ใช้

- **Next.js App Router** สำหรับเว็บแอปและ route หลัก
- **TypeScript** เพื่อความแม่นยำของข้อมูลและโค้ด
- **Supabase** สำหรับฐานข้อมูล Storage และ server-side admin access
- **Tailwind CSS v4** สำหรับ UI mobile-first
- **PromptPay QR** สำหรับการชำระเงิน
- **lucide-react** สำหรับ icon ในระบบ
- **AI photo flow** สำหรับประสบการณ์เสริมหลังการร่วมบุญ

## 10. ความปลอดภัยและข้อกำกับสำคัญ

- ไม่ยืนยันยอดร่วมทำบุญจาก AI หรือการอ่านภาพเพียงอย่างเดียว
- การยืนยันเงินต้องมีมนุษย์ตรวจสอบ
- Service role ของ Supabase ใช้เฉพาะ server-side
- ไม่เปิดเผยข้อมูลข้ามศูนย์ในหน้าที่เป็น center-scoped
- Admin กลางเน้นดูภาพรวมและกำกับดูแล ไม่ทำหน้าที่เป็นคิวปฏิบัติการรายงานศพแทนศูนย์
- หลีกเลี่ยงการลบข้อมูลหรือ action ทำลายข้อมูล หากไม่ได้รับคำสั่งชัดเจน

## 13. ฟลอการทำงานของเว็บแอพ

### ภาพรวมระบบ

เว็บแอพแบ่งการทำงานออกเป็น 4 บทบาทหลัก คือ ผู้ร่วมทำบุญ เจ้าภาพ ศูนย์ดูแลงาน และแอดมินกลาง โดยแต่ละบทบาทมีหน้าที่แยกกันชัดเจนเพื่อให้การรับร่วมทำบุญ การตรวจสลิป การพิมพ์ป้าย และการปิดงานเป็นระบบเดียวกัน

### 13.1 ฟลอเริ่มต้น: ศูนย์สร้างงาน

1. ศูนย์เข้าสู่ระบบ Dashboard ของศูนย์
2. กรอกข้อมูลผู้วายชนม์ เช่น ชื่อ รูป ชาตะ มรณะ อายุ และกำหนดการฌาปนกิจ
3. กรอกข้อมูลบัญชีหรือ QR สำหรับรับร่วมทำบุญ
4. ระบบสร้างหน้างานเฉพาะ เช่น `/{slug}`
5. ระบบสร้างลิงก์และ QR Code สำหรับแชร์ให้เจ้าภาพหรือวางหน้างาน

ผลลัพธ์ของฟลอนี้คือ งานศพพร้อมเปิดรับการร่วมบุญผ่านหน้าเว็บสาธารณะ

### 13.2 ฟลอผู้ร่วมทำบุญ: ร่วมมอบหรีดร่วมบุญ

1. ผู้ร่วมทำบุญเปิดลิงก์หน้างาน `/{slug}`
2. อ่านข้อมูลผู้วายชนม์ กำหนดการ และรายละเอียดงาน
3. กด CTA เพื่อไปหน้าโอนเงิน `/{slug}/payment`
4. ดู QR PromptPay หรือเลขบัญชี
5. โอนเงินผ่านแอพธนาคาร
6. แนบรูปสลิปและกรอกยอดเงิน
7. ระบบอัปโหลดสลิปและส่งผู้ร่วมทำบุญไปหน้ากรอกชื่อป้าย
8. ผู้ร่วมทำบุญกรอกชื่อ บุคคล/องค์กร และข้อความแสดงความอาลัย
9. ระบบบันทึกข้อมูลผู้ร่วมบุญไว้ในสถานะรอตรวจสอบ
10. ผู้ร่วมทำบุญได้รับหน้า e-card, ใบยืนยัน หรือภาพจำลองการมอบหรีด

จุดสำคัญของฟลอนี้คือ ผู้ร่วมทำบุญทำทุกอย่างผ่านมือถือได้ครบ ตั้งแต่โอนเงินจนถึงรับการ์ดขอบคุณ

### 13.3 ฟลอศูนย์: ตรวจสลิปและพิมพ์ป้าย

1. ศูนย์เปิด Dashboard ของงานนั้น
2. ดูรายการสลิปที่รอตรวจ
3. เปิดดูสลิป ยอดเงิน ชื่อผู้ร่วมทำบุญ และเวลาที่ส่งเข้าระบบ
4. เจ้าหน้าที่ตรวจสอบยอดเงินจริงกับบัญชีรับเงิน
5. หากถูกต้อง กดยืนยันสลิป
6. ระบบเปลี่ยนสถานะผู้ร่วมทำบุญเป็นยืนยันแล้ว
7. รายชื่อเข้าสู่คิวพิมพ์ป้าย
8. ศูนย์พิมพ์ป้ายชื่อผู้ร่วมบุญเพื่อติดหน้างาน
9. เมื่อติดป้ายแล้ว ศูนย์อัปเดตสถานะป้าย

จุดสำคัญคือ ระบบไม่ยืนยันเงินจาก AI หรือภาพสลิปอัตโนมัติ ต้องมีเจ้าหน้าที่ตรวจสอบก่อนเสมอ

### 13.4 ฟลอเจ้าภาพ: ดูสรุปงาน

1. เจ้าภาพเข้าสู่หน้า Dashboard ของเจ้าภาพ
2. ดูสรุปยอดร่วมทำบุญและจำนวนผู้ร่วมบุญ
3. ดูรายชื่อผู้ร่วมทำบุญและข้อความแสดงความอาลัย
4. ตรวจดูเอกสารหรือข้อมูลที่เกี่ยวข้องกับงาน
5. ใช้ข้อมูลสรุปสำหรับติดตามงานและปิดงานร่วมกับศูนย์

เจ้าภาพเน้นดูข้อมูลสรุปและรายชื่อ ไม่ต้องรับภาระตรวจสลิปหรือจัดการคิวปฏิบัติการเอง

### 13.5 ฟลอปิดงาน

1. ศูนย์ตรวจว่างานถึงกำหนดปิดหรือพิธีเสร็จสิ้นแล้ว
2. ตรวจสอบว่ายังมีสลิปรอตรวจหรือป้ายรอพิมพ์หรือไม่
3. หากยังมีรายการค้าง ระบบแจ้งให้ดำเนินการให้ครบก่อน
4. เมื่อทุกอย่างครบ ศูนย์ปิดงาน
5. ระบบล็อกไม่ให้รับสลิปใหม่
6. ระบบเก็บข้อมูลสำหรับรายงานและสรุปหลังงาน

ฟลอนี้ช่วยให้การปิดงานไม่ข้ามขั้นตอนสำคัญ เช่น สลิปค้างหรือป้ายที่ยังไม่ได้ติด

### 13.6 ฟลอแอดมินกลาง

1. แอดมินกลางเข้าสู่ Dashboard กลาง
2. ดูภาพรวมทุกศูนย์ เช่น จำนวนงาน ยอดร่วมทำบุญ รายการค้าง และสถานะระบบ
3. ดูรายงาน ESG และสถิติการลดขยะจากพวงหรีด
4. จัดการศูนย์ ผู้ใช้ สิทธิ์ และ audit log
5. ตรวจสอบความผิดปกติในภาพรวม เช่น สลิปรอตรวจสูงผิดปกติหรือศูนย์ที่ต้องติดตาม

แอดมินกลางทำหน้าที่กำกับดูแลภาพรวม ไม่ใช่คิวปฏิบัติการรายงานศพรายงานหนึ่งแทนศูนย์

### 13.7 Flow สำหรับทำสไลด์แบบย่อ

ศูนย์สร้างงาน → แชร์ลิงก์/QR → ผู้ร่วมทำบุญเปิดหน้างาน → โอนเงินและแนบสลิป → กรอกชื่อป้าย → ศูนย์ตรวจสลิป → ศูนย์พิมพ์ป้าย → ผู้ร่วมทำบุญรับ e-card → เจ้าภาพดูสรุป → ศูนย์ปิดงาน → แอดมินกลางดูภาพรวมและรายงาน ESG
