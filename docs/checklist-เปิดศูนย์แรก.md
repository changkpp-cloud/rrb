# ✅ Checklist ก่อนเปิดศูนย์แรก (หรีดร่วมบุญ)

เช็กให้ครบก่อนใช้งานจริงในงานศพแรก เรียงตามลำดับความสำคัญ

---

## 1. ฐานข้อมูล (Supabase) — ต้องทำก่อน
- [ ] **รัน migration คอลัมน์ยืนยันโอนเงิน** (ไม่งั้นยืนยันโอนเจ้าภาพไม่ได้)
  - Supabase Dashboard → **SQL Editor** → วาง + Run:
  ```sql
  ALTER TABLE memorials
    ADD COLUMN IF NOT EXISTS transfer_confirmed_at TIMESTAMPTZ,
    ADD COLUMN IF NOT EXISTS transfer_confirmed_by TEXT;
  ```
  - (ไฟล์: `supabase/migrations/20260624000000_add_transfer_confirmed.sql`)
- [ ] ตรวจ bucket storage มีครบ: `memorials`, `donations`

## 2. Environment Variables (Vercel + .env.local)
- [ ] `NEXT_PUBLIC_SUPABASE_URL` / `NEXT_PUBLIC_SUPABASE_ANON_KEY` / `SUPABASE_SERVICE_ROLE_KEY`
- [ ] `NEXT_PUBLIC_SITE_URL=https://ruamboon.online`
- [ ] `ADMIN_PASSWORD` (เปลี่ยนจาก default ESG2025)
- [ ] `HOST_SESSION_SECRET` (สำหรับ login เจ้าภาพ)
- [ ] `OPENAI_API_KEY` (สำหรับภาพ AI จำลองมอบหรีด)
- [ ] **`PRINTNODE_API_KEY`** (สำหรับพิมพ์ป้าย + ตรวจสถานะเครื่องพิมพ์) — ⚠️ ถ้าไม่ตั้ง ป้ายไม่พิมพ์ และแบนเนอร์เตือนเครื่องพิมพ์ทำงานไม่เต็มที่
- [ ] `MASTER_ACCESS_CODE` (รหัสเข้าทุก dashboard สำหรับทดสอบ)

## 3. ตั้งค่าเครื่องพิมพ์ (PrintNode)
- [ ] ติดตั้ง PrintNode client บนเครื่อง/คอมที่ต่อเครื่องพิมพ์หน้างาน
- [ ] เครื่องพิมพ์ขึ้น online ใน PrintNode + จด **Printer ID**
- [ ] กระดาษ A6 (148×105 มม.) พร้อม
- [ ] **ใส่ Printer ID ตอนเปิดงาน** (ฟอร์มเปิดงาน → ช่อง "เครื่องพิมพ์ป้ายชื่ออัตโนมัติ") — ถ้าไม่ใส่ = ป้ายไม่พิมพ์เงียบๆ

## 4. ตั้งค่าศูนย์ (Center)
- [ ] สร้างศูนย์ + ใส่ **เบอร์พร้อมเพย์** (`centers.phone`) — ใช้สร้าง QR ทุกงาน
- [ ] ใส่ข้อมูลบัญชีมูลนิธิ/ศูนย์รับเงิน
- [ ] รหัสเข้าศูนย์ (center_code) ใช้ได้

## 5. ทดสอบ flow ผู้ร่วมบุญ (เปิดงานทดสอบก่อน)
- [ ] เปิดงานทดสอบ → ได้ QR/ลิงก์
- [ ] เปิดลิงก์ใน **LINE** → เด้งออกเบราว์เซอร์อัตโนมัติ
- [ ] หน้าชำระเงิน: QR/เลขพร้อมเพย์/บันทึก QR/คัดลอกเลข ✓
- [ ] กดปุ่มแอปธนาคารที่ติดตั้ง → เปิดแอปได้ (เทสต์บนมือถือจริง)
- [ ] แนบสลิป + กรอกยอด → ส่งตรวจสลิป
- [ ] กรอกชื่อ + ยืนยันส่งพิมพ์ → เด้งหน้าขอบคุณเร็ว (~1 วิ)
- [ ] **ป้ายพิมพ์ออกจริงที่เครื่องหน้างาน**
- [ ] e-card + ภาพ AI จำลองมอบหรีด แสดงถูก

## 6. ทดสอบฝั่งศูนย์
- [ ] เห็นรายการผู้ร่วมบุญ + ยอด
- [ ] ป้าย error → กด "พิมพ์ซ้ำ" ได้
- [ ] ปิดเครื่องพิมพ์ → แบนเนอร์ "ตรวจสอบเครื่องพิมพ์" เด้งขึ้น (ศูนย์+เจ้าภาพ)
- [ ] **ปิดงาน** → ยอดสุทธิถูก (ยอดรวม − ราย×100)
- [ ] **ยืนยันโอนเงินเจ้าภาพ** (ต้องทำข้อ 1 ก่อน ไม่งั้นกดไม่ได้)

## 7. ทดสอบเจ้าภาพ
- [ ] login ด้วย host_code → เห็นรายชื่อ/ยอด/บัญชีรับเงิน
- [ ] กรอกบัญชีรับเงินเจ้าภาพได้

## 8. ทดสอบรายงาน
- [ ] หน้า report ศูนย์ (รายเดือน/ปี) → พิมพ์ PDF + CSV ได้

---

## หมายเหตุการตั้งค่าที่ค้นพบจากการทดสอบ
- **auto-print ทำเบื้องหลัง** (after) แล้ว — ผู้ร่วมบุญไม่ต้องรอ
- งานพิมพ์ที่ค้าง (เครื่องออฟไลน์) PrintNode พิมพ์ต่อเองเมื่อเครื่องกลับมา
- ค่าดำเนินการ **10% ของยอดร่วมบุญ** (เจ้าภาพได้สุทธิ 90%)
- ไม่มีขั้น "ติดบอร์ด" เป็นเงื่อนไข — พิมพ์แล้วถือว่าจบ
