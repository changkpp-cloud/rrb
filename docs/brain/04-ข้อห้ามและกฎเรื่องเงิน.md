# 4. ข้อห้ามและกฎเรื่องเงิน (สำคัญที่สุด)

> ⚠️ เป็น **กรอบหลักการเบื้องต้น** เพื่อสื่อสาร ไม่ใช่ความเห็นทางกฎหมาย/การคลังขั้นสุดท้าย — ต้องให้นิติกร/กองคลัง/ผู้รับผิดชอบของ อปท. รับรองก่อนดำเนินการจริง

## หลักการกระแสเงิน
1. ผู้ร่วมบุญโอนเงินเข้า **บัญชีของศูนย์บริหารหรีดร่วมบุญประจำ อปท.** (กลไกชุมชนที่ได้รับมอบหมาย) — **ไม่ใช่บัญชีรายได้ของ อปท.**
2. ระบบบันทึกรายการโอน + สลิป (auto-confirm)
3. ศูนย์หัก **ค่าดำเนินการ 10% ของยอดร่วมบุญ** เป็นค่ากระดาษ/หมึก/ดูแลอุปกรณ์/ประสานงานหน้างาน + กองทุนจิตอาสา
4. **ยอดสุทธิ = ยอดร่วมบุญรวม × 90%** (= ยอดรวม − 10%) → รวบรวมโอนให้เจ้าภาพหลังจบงาน
5. ศูนย์จัดทำรายงานสรุป: ยอดร่วมบุญ / ค่าดำเนินการ / ยอดนำส่งเจ้าภาพ
6. อปท. รับทราบผ่านรายงานของศูนย์ — **ไม่มีเงินร่วมบุญเข้าบัญชี อปท.**

## ข้อความที่ต้องพูดให้ชัด (กันความเข้าใจผิด)
- ❌ ไม่ใช่ "อปท. ทำธุรกิจพวงหรีด"
- ❌ ไม่ใช่ "เงินร่วมบุญเป็นรายได้ อปท."
- ❌ ไม่ใช่ "อปท. ตั้งงบเป็นเงินหมุนเวียน/เงินช่วยงานศพ"
- ✅ อปท. **สนับสนุนโครงสร้างพื้นฐาน + อุปกรณ์ + ระบบบริการสาธารณะ**
- ✅ การรับ-จ่ายเงินร่วมบุญ **ดำเนินการผ่านศูนย์/กลไกชุมชน**
- ✅ ทุกอย่าง **ตรวจสอบได้ มีรายงาน** (รองรับมุมมอง สตง.)

## งบประมาณ อปท. (โครงการนำร่อง)
- ใช้แผนงานที่เกี่ยวข้อง (สาธารณสุข/เคหะชุมชน/สร้างความเข้มแข็งชุมชน/สิ่งแวดล้อม)
- วงเงินนำร่อง **≤ 500,000 บาท** = บอร์ดใช้ซ้ำ, เครื่องพิมพ์, วัสดุเริ่มต้น, ป้าย/Standee, ระบบเว็บแอพ+คิวพิมพ์+รายงาน, ค่าอบรม
- **ไม่ใช่** งบจ่ายเงินช่วยงานศพ และ **ไม่ใช่** เงินหมุนเวียนในระบบร่วมบุญ

## ตัวเลขอ้างอิงในระบบ (ต้องตรงกันทุกฝั่ง)
- ค่าดำเนินการ: **10% ของยอดร่วมบุญ** (เจ้าภาพได้สุทธิ 90%) · สูตรกลาง `lib/fee.ts` (`FEE_RATE = 0.1`)
- ประมาณการลดขยะ: `1 พวงหรีด = 2 กก.`
