# 2. Flow ระบบ (สรุปสำหรับสื่อสาร)

> รายละเอียดเทคนิค/route/ตาราง ดูที่ `CLAUDE.md` — ไฟล์นี้สรุปเพื่อให้ AI agent อธิบาย/ทำคู่มือได้

## 4 ฝ่ายในระบบ
| ฝ่าย | คือใคร | ทำอะไร |
|---|---|---|
| **ผู้ดูแลส่วนกลาง (Admin)** | บริษัท/เจ้าของระบบ | ภาพรวมทั้งประเทศ, ศูนย์, รายงาน ESG/LPA, ตรวจสอบ |
| **ศูนย์บริหารประจำ อปท.** | จิตอาสา/ชุมชน/วัด | เปิดงาน, จัดคิวพิมพ์ป้าย, ปิดงาน, โอนเงินเจ้าภาพ, รายงานส่ง อปท. |
| **เจ้าภาพ** | ครอบครัวผู้วายชนม์ | ดูรายชื่อ/ยอด, รับเงินที่รวบรวมได้ |
| **ผู้ร่วมบุญ** | แขกร่วมงาน | ร่วมบุญเป็นเงิน + ฝากชื่อขึ้นป้าย |

## ขั้นตอนผู้ร่วมบุญ (Donor flow — 4 หน้า)
1. **หน้างาน** — ข้อมูลผู้วายชนม์ + กำหนดพิธี
2. **ชำระเงิน** — สแกน QR พร้อมเพย์ของศูนย์ / กดเข้าแอปธนาคาร → โอน → แนบสลิป
3. **ป้ายชื่อ** — กรอกชื่อ/หน่วยงาน/ข้อความอาลัยที่จะขึ้นป้าย
4. **ขอบคุณ** — e-card อนุโมทนา + ภาพ AI จำลองมอบหรีด + ใช้เป็นหลักฐานการร่วมบุญ

## หัวใจระบบ
- **Auto-confirm + Auto-print** — แนบสลิป+กดยืนยัน → บันทึกเงินทันที + ส่งคิวพิมพ์ป้ายอัตโนมัติ (ศูนย์ไม่ต้องนั่งตรวจสลิปทีละใบ)
- **ศูนย์ = ผู้ดูแล/จัดการหน้างาน** (พิมพ์ป้าย ติดบอร์ด ปิดงาน โอนเจ้าภาพ) ไม่ใช่ด่านอนุมัติเงิน
- **วันฌาปนกิจ** — ศูนย์รวบรวมเงินสุทธิโอนให้เจ้าภาพ + ปิดงาน
- **รายงาน** — ศูนย์ออกรายงานรายเดือน/รายปีส่ง อปท. (จำนวนงาน, ยอดร่วมบุญ, ค่าดำเนินการ, นำส่งเจ้าภาพ, ลดขยะ)

## ช่องทางเข้าถึง
- เปิดผ่าน QR/ลิงก์ประจำงาน (เช่น `ruamboon.online/evt-xxxx`)
- เปิดจากแชท (LINE/FB) ระบบจะพาออกไปเบราว์เซอร์จริงให้ deep link แอปธนาคาร + บันทึกรูปทำงานได้
