## ✅ ฟีเจอร์ที่เสร็จแล้วและใช้งานได้

### Donor flow (ผู้ร่วมบุญ) — ครบทั้งเส้น
- หน้าแรก: ลิสต์งานเปิดอยู่ + **บอร์ดรำลึก** ป้ายผู้ร่วมบุญรวมทุกงาน (ผู้มอบจริงขึ้นก่อน เติมด้วยป้ายสปอนเซอร์ กดป้ายดูข้อความหลังป้าย)
- หน้างาน → ชำระเงิน (QR พร้อมเพย์จากเบอร์เจ้าภาพที่ยืนยัน OTP + แสดงเลขบัญชี/เลขพร้อมเพย์พร้อมปุ่มคัดลอกแยก + deep link แอปธนาคาร + เด้งออกจาก in-app browser LINE/FB)
- อัปโหลดสลิป (JPG/PNG เท่านั้น + hash ตรวจสลิปซ้ำแบบเตือนย้อนหลัง)
- กรอกป้ายชื่อ: ชื่อ + ตำแหน่ง (หน้าป้าย) + **ข้อความอาลัย/กำลังใจ (หลังป้าย)** + preview ป้ายสองด้านก่อนยืนยัน
- **Auto-confirm + auto-print**: donation ยืนยันทันที + ยิงพิมพ์ป้ายผ่าน PrintNode อัตโนมัติ
- หน้าขอบคุณ: e-card + **ภาพ AI จำลองมอบหรีด** (gpt-image-1.5) + แชร์ลิงก์รับรูปภายหลัง (`/ai-photo/jobs/[id]`)
- รองรับรูป iPhone (HEIC → JPEG อัตโนมัติผ่าน heic2any) ทุกจุดอัปโหลด · เจนภาพ AI **ไม่จำกัดจำนวน** (ปลดลิมิตฟรี 1 ภาพแล้ว)

### ศูนย์ (Center)
- เปิดงาน: บังคับกรอกบัญชีเจ้าภาพครบ 3 ช่อง + ยืนยันเบอร์ด้วย OTP ก่อนเปิด
- จัดการงาน + **แก้บัญชี/เบอร์ภายหลังได้ ต้องยืนยัน OTP ทุกครั้ง** (commit ค่าใหม่พร้อม OTP)
- ปิดงาน / งานวันนี้ (operations: KPI + คิวป้าย + เตือนสลิปซ้ำ) / เก็บค่าดำเนินการ-คืนบอร์ด (transfers)
- รายงานรายเดือน/รายปีส่ง อปท. (PDF/CSV) + compliance tracker (ทำเครื่องหมายส่งแล้ว)
- ล็อกอินด้วยเบอร์มือถือ + รหัสผ่าน (แอดมินกลางออกรหัสให้)

### อปท. (lgo_observer)
- `/oversight` (บ้าน read-only: KPI/การเงินโปร่งใส/สิ่งแวดล้อม — ไม่มี PII) + `/compliance` + `/report`

### เจ้าภาพ (Host)
- แดชบอร์ด: ยอดรวม/รายชื่อ/รายงาน/บัญชี/ภาพ AI + แก้ไขงาน + สิทธิ์หมดอายุ 30 วันหลังปิดงาน + เตือนเครื่องพิมพ์ออฟไลน์

### แอดมินกลาง
- overview/analytics/ESG/รายงานภูมิศาสตร์/system health/**audit log viewer** (ดูบันทึกเรื่องเงิน 200 รายการล่าสุด)/จัดการศูนย์/ออกรหัสเข้าศูนย์/แบนเนอร์/AI prompts

### โครงสร้างพื้นฐาน
- **Audit log เรื่องเงิน**: แก้บัญชี (verify_host_bank) / ปิดงาน (close_memorial) / เก็บค่าดำเนินการ (confirm_transfer) — เก็บผู้ทำ/role/ค่าเก่า-ใหม่/IP
- **SMS ผ่าน ThaiBulkSMS** (โค้ดพร้อม): OTP 2 จุด + แจ้งศูนย์เมื่อป้ายพิมพ์ไม่สำเร็จ — ตั้ง env `THAIBULKSMS_*` ครบ = ส่งจริง / ไม่ตั้ง = โหมดทดสอบโชว์รหัสบนจอ (**สถานะปัจจุบัน: โหมดทดสอบ** — ตัดสินใจใช้จำลองไปก่อนจนเปิดศูนย์แรก ~1,500 บาท/ปี)
- **นโยบายแจ้งเตือน**: ไม่ส่ง SMS เจ้าภาพรายคน (เงินมีแจ้งเตือนธนาคาร + รายชื่อดูแดชบอร์ด) · แจ้งเฉพาะศูนย์เมื่อป้ายพิมพ์ล้ม
- **หน้า PDPA** `/privacy` (ผู้ควบคุมข้อมูล: บริษัท หรีดร่วมบุญ จำกัด + ศูนย์/อปท. เป็นผู้ควบคุมร่วม · ติดต่อ rrbzerowaste@gmail.com / 096-235-4999)
- ระบบพิมพ์: PrintNode ค้างคิวเมื่อเครื่องออฟไลน์ + ปุ่มพิมพ์ซ้ำ + printer status alert
- คิวงานเบื้องหลัง (outbox_jobs + worker cron) + สถิติ denormalized (ceremony_stats/tenant_stats)

## ⏳ ฟีเจอร์ที่ทำค้าง / ยังไม่เสร็จ

| เรื่อง | สถานะ |
|---|---|
| **ตรวจสลิปจริง** (EasySlip/SlipOK) | ยังไม่ทำ — ปัจจุบัน auto-confirm โดยไม่ตรวจว่าสลิปจริง/ยอดตรง (จุดเสี่ยงหลัก, จัดว่า "คุ้มสุด" ที่ควรทำต่อ) |
| **เปิด SMS จริง** | โค้ดพร้อมแล้ว รอสมัคร ThaiBulkSMS + ใส่ env 3 ตัวใน Vercel (คู่มือ: `docs/setup-sms-otp.md`) — ตัดสินใจเลื่อนไว้ก่อน |
| **แยกเบอร์ OTP ↔ เบอร์พร้อมเพย์** | ตกลงแนวทางแล้ว (OTP = เบอร์คนแจ้ง, พร้อมเพย์ = อีกเบอร์) แต่ยังไม่ implement — ต้องเคาะว่าเบอร์พร้อมเพย์ต้อง OTP ด้วยไหม |
| พร้อมเพย์แบบเลขบัตรประชาชน | ยังไม่รองรับ (ตอนนี้ผูกเบอร์มือถือเท่านั้น) |
| ป้ายสปอนเซอร์บอร์ดรำลึก — หน้าแอดมินจัดการ | ยังแก้ได้เฉพาะในโค้ด (`sponsorTags` ใน `components/DynamicVirtualBoard.tsx`) — เฟส 3 ที่วางไว้ |
| เอฟเฟกต์ "พลิกป้าย" บนบอร์ด | ปัจจุบันเป็น modal เด้ง — flip เป็นออปชันที่คุยไว้ ยังไม่ทำ |
| เทมเพลต export LPA/ITA/จังหวัดสะอาด + มุมรวมหลายศูนย์ | ค้างจากแผน IAM (ระบุใน CLAUDE.md) |
| ระบบ login แบบ proper (JWT) | ยังใช้ cookie + password/OTP แบบง่าย |
| Payment webhook (`/api/webhooks/payment` + ตาราง payments) | โครงพร้อมแต่ยังไม่ต่อ provider จริง |
| Error monitoring (Sentry) / backup อัตโนมัติ | ยังไม่มี |

## ⚠️ Known issues / ข้อจำกัดที่รู้อยู่

1. **ยอดเงิน donation เชื่อจากผู้กรอก** — ไม่มีการตรวจกับสลิปจริง → ยอดรวม/ค่าดำเนินการ 10% อาจคลาดจากความจริงได้ (จะหายเมื่อทำตรวจสลิป)
2. **ภาพ AI ที่เจนก่อน 30 มิ.ย. 2569 แตก** — URL เก่าชี้ bucket `donations` (private) ก่อนย้ายมาเก็บใน `memorials` (public) → ต้องเจนใหม่เท่านั้น
3. **OTP โหมดทดสอบโชว์รหัสบนจอ** — ใครก็ยืนยันได้เพราะเห็นรหัส (ตั้งใจ ใช้ชั่วคราวก่อนเปิด SMS จริง) และกล่องยังเขียน "โหมดทดสอบ" ซึ่งผู้ใช้จริงอาจงง
4. **การส่งภาพ AI ต้องมี env ครบ** — `OPENAI_API_KEY` + องค์กร OpenAI ต้อง verified ถึงใช้ gpt-image ได้ (เคยเป็นสาเหตุเจนล้ม)
5. `center_staff`/`center_viewer` ยังอยู่ใน enum แต่ไม่ออกใหม่ (backward-compat)
6. ตาราง `ai_photo_credits` ถูกเลิกใช้แต่ยังอยู่ในฐานข้อมูล (ไม่มีผลต่อระบบ)
7. เอกสารสรุปยุคเก่าที่ root (`SYSTEM_DOCUMENTATION.md`, `project-status-rrb.md`, `ระบบสรุป-หรีดร่วมบุญ.md`) อาจล้าสมัยกว่าโค้ด — ยึด `CLAUDE.md` + `CHANGELOG.md` + โฟลเดอร์นี้เป็นหลัก
8. มีการทำงานคู่ขนานหลาย session ในอดีต ทำให้เคยเกิดงานซ้ำ/แนวทางต่างกันเล็กน้อย (เช่น บอร์ดรำลึกมี 2 implementation ก่อน merge เหลือ `DynamicVirtualBoard`)
