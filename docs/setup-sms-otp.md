# คู่มือเปิดใช้งาน SMS OTP (ThaiBulkSMS)

ระบบส่งรหัส OTP ยืนยันเบอร์เจ้าภาพ รองรับ **ThaiBulkSMS** อยู่แล้วในโค้ด (`lib/sms.ts`)
ตอนนี้ยังเป็น **โหมดทดสอบ** (โชว์รหัสบนจอ) จนกว่าจะตั้งค่า env ครบ 3 ตัว

> ⚠️ บัญชี ThaiBulkSMS + เครดิต ต้องสมัคร/เติมเอง (ใช้ข้อมูล/บัตร/เบอร์ของผู้ดูแล) — โค้ดทำส่วนนี้แทนไม่ได้

---

## ขั้นที่ 1 — สมัคร ThaiBulkSMS
1. ไปที่ https://www.thaibulksms.com → สมัครสมาชิก (ใช้อีเมล + เบอร์มือถือ)
2. ยืนยันตัวตน (OTP เข้ามือถือ) ให้เรียบร้อย

## ขั้นที่ 2 — เติมเครดิต
- เลือกแพ็กเกจ (คิดรายข้อความ ~0.10–0.29 บาท/ครั้ง แล้วแต่ปริมาณ)
- OTP ภาษาไทยสั้น ๆ = 1 เครดิต/ข้อความ

## ขั้นที่ 3 — จดทะเบียนชื่อผู้ส่ง (Sender Name)
- เมนู Sender Name → ขอชื่อ เช่น `RuamBoon` (ปกติฟรี รออนุมัติ)
- **ต้องได้รับอนุมัติก่อน** ถึงจะส่งจริงได้ · ชื่อนี้ = ค่า `THAIBULKSMS_SENDER`

## ขั้นที่ 4 — เอา API Key / Secret
- เมนู API / Developer → คัดลอก **API Key** และ **API Secret**

## ขั้นที่ 5 — ใส่ค่าใน Vercel (อย่าใส่ในโค้ด/อย่า commit)
Vercel → โปรเจกต์ → **Settings → Environment Variables** → เพิ่ม 3 ตัว:

```
THAIBULKSMS_API_KEY=<API Key ที่คัดลอกมา>
THAIBULKSMS_API_SECRET=<API Secret ที่คัดลอกมา>
THAIBULKSMS_SENDER=RuamBoon
```

จากนั้น **Redeploy** (หรือ deploy รอบถัดไป)

---

## พฤติกรรมระบบ
- **ตั้งครบ 3 ตัว** → ส่ง SMS จริง, ไม่โชว์รหัสบนจอ · ส่งไม่สำเร็จจะเด้ง error บอกสาเหตุ
- **ยังไม่ตั้ง / ตั้งไม่ครบ** → โหมดทดสอบ: โชว์รหัส OTP บนจอ (กล่องเหลือง) + log ฝั่ง server

## ทดสอบหลังตั้งค่า
1. เปิดงานใหม่ (หรือหน้าจัดการงาน → การเงิน → แก้บัญชี) → กด "ส่งรหัส OTP"
2. ตรวจว่ามี SMS เข้ามือถือเบอร์เจ้าภาพ → กรอกรหัส → ยืนยัน

## แก้ปัญหาเบื้องต้น
- **ส่งไม่ออก / error** : เช็กว่า Sender Name อนุมัติแล้ว, เครดิตเหลือ, key ถูกต้อง
- **ยังโชว์รหัสบนจอ** : แปลว่ายังอ่าน env ไม่ครบ 3 ตัว — ตรวจการสะกดชื่อ env + กด Redeploy
